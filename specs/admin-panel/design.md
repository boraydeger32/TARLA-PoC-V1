## 1. Architectural Principles

### 1.1 Layered Architecture (MVC-flavored)

Alpine.js is not a component framework like React вҖ” it is a progressive-enhancement runtime sprinkled over static HTML. To keep this codebase sane for a non-trivial admin panel, we enforce **classical MVC-style separation** through folder layering:

| Layer | Responsibility | Characteristics |
|---|---|---|
| **Models** | Domain entities + DTO schemas | zod schemas matching Pydantic; JSDoc typedefs |
| **Views** | HTML templates + per-page JS modules | Alpine directives only; no business logic |
| **Controllers** | Alpine data factories that orchestrate page logic | Form submission flow, multi-step wizards, coordinated actions |
| **Queries** | `@tanstack/query-core` wrappers around services | `getListings()`, `getListing(id)`, `approveListing()` |
| **Services** | OOP classes that talk to API | `BaseService` вҶ’ `ListingsService`, `UsersService`. Singleton instances exported. |
| **Stores (Alpine.store)** | Global client-only state | auth, theme, locale, ui (sidebar state, drawer open, etc.) |
| **API** | axios client + interceptors + endpoints | Low-level HTTP, token refresh, error normalization |

### 1.2 Separation: Alpine Stores vs Query Cache

- **Alpine stores** вҶ’ Client-owned state (auth tokens, theme, locale, sidebar collapsed, modal stack)
- **@tanstack/query-core** вҶ’ Server-owned state (listings, users, anything from `/api/v1/*`)
- **NEVER** store API response data in Alpine stores. **NEVER** store UI state in the query cache.

### 1.3 Backend-Agnostic Contract

The `services/` layer is the **only** layer allowed to know about HTTP. Views, controllers, and query wrappers SHALL NOT import `axios` directly. This enables the MSWвҶ’FastAPI swap without any refactor.

```javascript
import { errorInterceptor } from './interceptors/errorInterceptor.js';
import { getEnv } from '@/utils/env.js';

/**
 * @typedef {import('axios').AxiosInstance} AxiosInstance
 * @typedef {import('axios').AxiosRequestConfig} AxiosRequestConfig
 */

export class ApiClient {
  static #instance = null;

  /** @type {AxiosInstance} */
  axios;

  constructor() {
    if (ApiClient.#instance) {
      throw new Error('Use ApiClient.getInstance()');
    }
    this.axios = axios.create({
      baseURL: getEnv('VITE_API_BASE_URL'),
      timeout: 15000,
      headers: { 'Content-Type': 'application/json' },
    });
    this.#applyInterceptors();
  }

  static getInstance() {
    if (!ApiClient.#instance) {
      ApiClient.#instance = new ApiClient();
    }
    return ApiClient.#instance;
  }

  #applyInterceptors() {
    this.axios.interceptors.request.use(authInterceptor);
    this.axios.interceptors.response.use((r) => r, refreshInterceptor);
    this.axios.interceptors.response.use((r) => r, errorInterceptor);
  }

  /** @template T @param {string} url @param {AxiosRequestConfig} [config] @returns {Promise<T>} */
  get(url, config) { return this.axios.get(url, config).then((r) => r.data); }

  post(url, body, config) { return this.axios.post(url, body, config).then((r) => r.data); }
  put(url, body, config) { return this.axios.put(url, body, config).then((r) => r.data); }
  patch(url, body, config) { return this.axios.patch(url, body, config).then((r) => r.data); }
  delete(url, config) { return this.axios.delete(url, config).then((r) => r.data); }
}

export const apiClient = ApiClient.getInstance();
```

```javascript
// src/queries/listingsQueries.js
import { QueryObserver } from '@tanstack/query-core';
import { queryClient } from './queryClient.js';
import { queryKeys } from './queryKeys.js';
import { listingsService } from '@/services/ListingsService.js';

/**
 * Creates an observer for a listings list query. The Alpine component calls
 * `observer.subscribe(cb)` and unsubscribes on destroy.
 */
export function listingsListObserver(params) {
  return new QueryObserver(queryClient, {
    queryKey: queryKeys.listings.list(params),
    queryFn: () => listingsService.list(params),
  });
}

export function listingDetailObserver(id) {
  return new QueryObserver(queryClient, {
    queryKey: queryKeys.listings.detail(id),
    queryFn: () => listingsService.getById(id),
  });
}

/**
 * Mutation helpers вҖ” call directly, invalidate on success.
 */
export async function approveListing(id) {
  const result = await listingsService.approve(id);
  queryClient.invalidateQueries({ queryKey: queryKeys.listings.all });
  return result;
}

export async function rejectListing(id, reason) {
  const result = await listingsService.reject(id, reason);
  queryClient.invalidateQueries({ queryKey: queryKeys.listings.all });
  return result;
}
```

### 3.5 Alpine Component Wiring for Queries

```javascript
// src/views/listings/list.js
import { listingsListObserver } from '@/queries/listingsQueries.js';

export function listingsListPage() {
  return {
    items: [],
    total: 0,
    loading: true,
    error: null,
    params: { page: 1, size: 25, status: 'all' },

    init() {
      this.observer = listingsListObserver(this.params);
      this.unsubscribe = this.observer.subscribe((result) => {
        this.items = result.data?.items ?? [];
        this.total = result.data?.total ?? 0;
        this.loading = result.isLoading;
        this.error = result.error;
      });
    },

    destroy() {
      this.unsubscribe?.();
    },

    updateParams(next) {
      this.params = { ...this.params, ...next };
      this.observer.setOptions({
        ...this.observer.options,
        queryKey: ['listings', 'list', this.params],
      });
    },
  };
}
```

## 9. MSW вҶ’ FastAPI Swap Contract (The Sacred Invariant)

This is the **most important architectural invariant** of the project. If this breaks, Phase 2 (backend integration) becomes a refactor nightmare.

### 9.1 Rules

1. **Service methods SHALL NOT know about MSW.** MSW is a transparent HTTP intercept layer. Services only see `axios`.
2. **All URLs SHALL be relative**, resolved against `VITE_API_BASE_URL`. Never hardcode `http://localhost:...` in services.
3. **Mock response shapes SHALL exactly match the agreed FastAPI response shape** вҖ” including pagination wrapper, timestamp format (ISO 8601), field naming (snake_case from backend matching DTO names).
4. **MSW worker SHALL start conditionally:**
   ```javascript
   // src/main.js
   async function enableMocks() {
     if (import.meta.env.VITE_USE_MOCKS !== 'true') return;
     const { worker } = await import('./mocks/browser.js');
     await worker.start({ onUnhandledRequest: 'warn' });
   }
   enableMocks().then(() => bootApp());
   ```
5. **Error responses SHALL match FastAPI's standard error format:**
   ```json
   { "detail": [{ "loc": [...], "msg": "...", "type": "..." }] }
   ```
   OR
   ```json
   { "detail": "Error message string" }
   ```

### 9.2 Environment Variables

```
# .env.development (mock mode)
VITE_API_BASE_URL=http://localhost:5173/api/v1
VITE_USE_MOCKS=true

# .env.production (live mode)
VITE_API_BASE_URL=https://api.aperant.com/api/v1
```

## 11. Form Handling Convention

All forms use **Alpine `x-model` + zod**:

```javascript
// src/views/listings/edit.js
import { z } from 'zod';
import { i18n } from '@/i18n/index.js';

const schema = z.object({
  title: z.string().min(5),
  price: z.number().positive(),
});

export function listingEditForm() {
  return {
    form: { title: '', price: 0 },
    errors: {},

    validate() {
      const result = schema.safeParse(this.form);
      this.errors = result.success
        ? {}
        : Object.fromEntries(
            result.error.issues.map((i) => [i.path.join('.'), i18n.t(`forms.validation.${i.code}`)])
          );
      return result.success;
    },

    async submit() {
      if (!this.validate()) return;
      // call controller -> query wrapper -> service
    },
  };
}
```

HTML:

```html
<form x-data="listingEditForm" @submit.prevent="submit">
  <div>
    <label x-t="'listings.form.title'"></label>
    <input x-model="form.title" class="flowbite-input-classes" />
    <p class="text-red-500" x-text="errors.title" x-show="errors.title"></p>
  </div>
  <!-- ... -->
</form>
```

Shared form field factories live in `src/components/forms/`.

## 12. Error Handling

- **Query errors:** Surfaced through the observer's `error` field вҶ’ the Alpine page reads it вҶ’ shows a Flowbite toast via `toast.error(...)`.
- **Global error boundary:** Wrap route mounting in a try/catch; on failure, show the `errors/500.html` template.
- **Form errors:** zod issues, mapped to i18n keys, displayed via `x-text` under each field.
- **API errors:** Normalized by `errorInterceptor` into `{ code, message, fields? }` shape.

## 17. Phase 2 Hand-off Checklist (for future)

When backend is ready, switching to live API requires:

1. Set `VITE_USE_MOCKS=false` in `.env.production`
2. Set `VITE_API_BASE_URL=https://<actual-host>/api/v1`
3. Verify FastAPI response shapes match `models/*ResponseSchema` zod schemas
4. Remove or keep `src/mocks/` for dev-time work (can still run in parallel)
5. No changes expected in `views/`, `controllers/`, `queries/`, `services/`, `stores/`

If any of step 3's shapes mismatch, it is a **FastAPI implementation bug** вҖ” not a frontend refactor.
